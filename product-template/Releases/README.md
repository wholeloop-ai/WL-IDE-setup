# Releases

Notas de release orientadas a **producto y operaciones** (qué cambió, para quién, cómo validar). No sustituyen handoffs ADWF ni artefactos en `Features/`.

## Convención de nombre

```
YYYY-MM-DD-<epic-or-feature-slug>.md
```

Ejemplo: `2026-06-01-epic-checkout-v2.md`

## Contenido mínimo

1. **Resumen** — 2–4 frases para stakeholders
2. **Tickets** — Linear (epic + hijos)
3. **Repos y PRs** — enlaces a merge `main`
4. **Deploy** — Supabase, Railway, Vercel (checklist)
5. **Validación** — smoke tests post-deploy
6. **Fuera de alcance / siguiente**

Handoff operativo detallado: `Progress/adwf-handoffs/`.
